# COMP3011 Search Engine Tool

A command-line search engine built for the COMP3011 Web Services and Web Data module.

## Overview

This tool crawls [quotes.toscrape.com](https://quotes.toscrape.com), builds an inverted
index of all words found across the site, and lets you query it interactively via a
command-line shell.

## Architecture

The project is split into three independent modules that are wired together by the CLI:

- `crawler.py` - visits pages using a BFS queue, extracts same-domain links, and
  returns a dict of URL to raw HTML
- `indexer.py` - parses each HTML page, tokenises the text, and builds an inverted
  index storing word frequency and positions per URL
- `search.py` - accepts print and find queries against the index and returns
  formatted results
- `main.py` - the interactive shell that connects all three modules

## Design Decisions

**Inverted index structure**

The index is a nested dict:

    word -> { url -> { frequency: int, positions: [int] } }

This gives O(1) lookup by word and O(1) lookup of which URLs contain that word.
Storing positions as well as frequency means the index can support more advanced
features like phrase matching in future.

**BFS crawling**

A breadth-first queue was chosen so pages are discovered level by level outward
from the start URL. URLs are normalised (fragments stripped, trailing slashes
removed) before being added to the visited set, which prevents the same page
being fetched twice under slightly different URLs.

**AND search for multi-word queries**

The find command computes the set intersection of URLs across all query terms.
This means a query like "good friends" only returns pages containing both words,
which gives more precise results than OR search would.

**JSON persistence**

The index is saved as JSON. It is human-readable, requires no extra dependencies,
and loads fast enough for a dataset this size.

**Case insensitivity**

All tokens are lowercased at index build time rather than at query time. This means
the index itself is always lowercase and queries work correctly regardless of how
the user types them.

## Setup

Install dependencies:

    pip install -r requirements.txt

For development and running tests, also install:

    pip install -r requirements-dev.txt

## Usage

Start the interactive shell:

    python src/main.py

### Commands

    build                crawl the website and build the inverted index
    load                 load a previously built index from disk
    print <word>         print all index entries for a word
    find <query>         find pages containing all words in the query
    help                 show available commands
    quit                 exit the tool

### Examples

    > build
    > load
    > print nonsense
    > find indifference
    > find good friends

## Testing

Run the full test suite:

    python -m pytest tests/ -v

Run with coverage:

    python -m pytest tests/ -v --cov=src --cov-report=term-missing
